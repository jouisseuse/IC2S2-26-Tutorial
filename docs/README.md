# Documentation

This folder contains conceptual documentation for the framework.

The docs should read like a practical methodological guide for psychologists and behavioral scientists, not like internal engineering notes.

## Who Should Use This

Use this folder if you want to understand the concepts behind multiplayer experiments, choose between human-only, LLM-only, and mixed designs, or explain the framework in a paper or tutorial.

## What Not To Put Here

Do not put runnable code, raw data, API keys, or exercise-only instructions here. Code belongs in `engines/` or `shared/`, data belongs in `analysis/sample_data/`, and live exercises belong in `hands_on/`.

## Completion Standard

A behavioral scientist should be able to read these docs and understand why multiplayer experiments need clear choices about agents, actions, information, interaction, outcomes, and analysis.
